# 1️⃣ PROJECT OVERVIEW

The **Restaurant POS (Point of Sale) System** is a full-scale restaurant management solution designed to digitize and automate daily restaurant operations.

### ✅ Core Functionalities

* Restaurant Authentication (Separate System Owner Login)
* Employee Authentication (Role-Based Login)
* Staff Management
* Table Management
* Menu & Category Management
* Inventory Tracking
* Order Processing (Real-time)
* Kitchen Workflow Handling
* Billing & Payments
* Password Reset System (NEW)

---

### 🛠️ Technology Stack

* **Backend:** Node.js + Express
* **Frontend:** React.js
* **Database:** SQL Server
* **Architecture:** Modular (Schema-based)
* **Security:** JWT Authentication + Role-Based Access

---

### 🗂️ Database Schema Structure

* **auth** → Authentication
* **master** → Core restaurant data
* **orders** → Orders & Items
* **kitchen** → Kitchen workflow
* **billing** → Bills & payments

# 3️⃣ 🔑 RESET PASSWORD FUNCTIONALITY (NEW)

## 📌 Applies To:

* Restaurant (Owner)
* Employees (All Roles)

---

## 🔄 Reset Password Flow

### Step 1: Forgot Password

User clicks **"Forgot Password"**

* Enter Email
* System verifies user

---

### Step 2: Token Generation

* Backend generates **JWT Reset Token**
* Token has expiry (e.g., 15 minutes)

---

### Step 3: Email Sent

* Reset link sent via email

```
http://frontend/reset-password/:token
```

---

### Step 4: Reset Password Page

User:

* Enters new password
* Confirms password

---

### Step 5: Backend Validation

* Token verified
* Password hashed (bcrypt)
* Database updated

---

### Step 6: Success

* User can login with new password

---

## 🔐 Security Features

* Token expiry
* One-time use token
* Password hashing
* Email verification

# 5️⃣ COMPLETE SYSTEM FLOW (UPDATED 🔥)

---

## 🟢 STEP 1: Restaurant Login (MAIN ENTRY)

* Restaurant Owner logs in
* Gets full admin access

---

## 🟢 STEP 2: Initial Setup (ADMIN)

Admin performs:

* Add Categories

* Add Tables
* Add Inventory
* ✅ Add Employees (IMPORTANT)

---

## 🟢 STEP 3: Employee Creation Flow

Admin:

* Creates employees
* Assigns roles:

  * Waiter / Kitchen / Cashier

System:

* Stores in `auth.Users`

---

## 🟢 STEP 4: Employee Login (SEPARATE LOGIN PAGE)

🔥 Important Feature:

* Employees login from **different login page**

Example:

* `/login` → Restaurant
* `/employee/login` → Staff

---

## 🟢 STEP 5: Customer Arrival

Waiter:

* Selects available table
* Adds customer details
* Creates order

---

* Set quantity
* Click **Send to Kitchen**

---

## 🟢 STEP 7: Database Operations

* `orders.Orders` → Created
